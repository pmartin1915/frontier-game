## 1. ROADMAP Phase 1 — Condition Escalation (Item 2)
Severity: High
Status: Missing Implementation

### Findings
- The `ActiveCondition` type in `src/types/game-state.ts` includes a `daysUntilCritical` field that is never decremented.
- The `calculateHealth` function in `src/systems/health.ts` does not process `daysUntilCritical` or emit "worsened" events when it reaches zero.
- No permanent debuffs are applied when conditions escalate to critical.

### Recommendations
- Extend `calculateHealth` to:
  1. Decrement `daysUntilCritical` for each active condition.
  2. When `daysUntilCritical` reaches zero, emit a "worsened" `HealthEvent`.
  3. Apply a permanent debuff (e.g., broken bone → lasting movement penalty).
- Add unit tests in `tests/systems/health.test.ts` to verify escalation logic.

---

## 2. ROADMAP Phase 1 — Companion Loyalty Momentum (Item 3)
Severity: Medium
Status: Missing Implementation

### Findings
- The `processCompanionDay` function in `src/systems/companions.ts` does not implement a loyalty-dependent desertion roll.
- Desertion probability is currently only a function of low morale and low loyalty, without momentum or hysteresis.

### Recommendations
- Add a loyalty-dependent desertion roll in `processCompanionDay`:
  - Base desertion chance = `max(0, (15 - loyalty) / 100) * (1 - morale / 100)`.
  - Apply a small hysteresis bonus (e.g., +5 loyalty) if the companion stayed the previous day.
- Extend `tests/systems/companions.test.ts` to verify the new desertion logic.

---

## 3. ROADMAP Phase 1 — Hunting Yield by Biome/Season (Item 4)
Severity: Medium
Status: Missing Implementation

### Findings
- The `calculateSupplyConsumption` function in `src/systems/supplies.ts` returns a flat yield for hunting.
- No biome or season factor is applied to hunting success or yield.

### Recommendations
- Add a `HUNTING_YIELD_BY_BIOME` table in `src/systems/supplies.ts`.
- Add a date-based season factor (e.g., ±20% randomization).
- Randomize yield ±20% around the biome/season base.
- Extend `tests/systems/supplies.test.ts` to verify biome/season effects.

---

## 5. Code Quality — Magic Numbers in Health System
Severity: Low
Status: Present

### Findings
- `src/systems/health.ts` contains multiple magic numbers (e.g., `CRISIS_HEALTH_THRESHOLD = 40`, `CRISIS_RECOVERY_PER_POINT = 0.05`).
- These constants are not documented or grouped for maintainability.

### Recommendations
- Group all magic numbers into a `HEALTH_CONSTANTS` object at the top of the file.
- Add JSDoc comments explaining the purpose and scaling of each constant.

---

## 6. Code Quality — Redundant RNG Injection
Severity: Low
Status: Present

### Findings
- All system calculators (`movement.ts`, `health.ts`, `supplies.ts`, `companions.ts`) accept an optional `rng` parameter for testing.
- The pattern is repeated in each file, increasing boilerplate.

### Recommendations
- Extract the RNG injection pattern into a shared utility type:
  ```ts
  type WithRng<T> = T & { rng?: () => number };
  ```
- Apply the type to all system input interfaces.
