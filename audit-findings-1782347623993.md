*Location:* `src/systems/supplies.ts`
*Status:* Open
*Description:* Hunt yields use flat values without biome/season variation.
*Impact:* Reduces game realism and strategic depth in resource management.
*Evidence:* Flat yield values observed in code review (lines 42-48)
*Recommendations:*
```typescript
const HUNTING_YIELD_BY_BIOME = {
  forest: { base: 3, seasonFactor: 0.3 },
  desert: { base: 1, seasonFactor: 0.1 },
  // ... other biomes
};

function calculateHuntYield(biome: Biome, dayOfYear: number): number {
  const biomeData = HUNTING_YIELD_BY_BIOME[biome];
  const seasonFactor = 1 + biomeData.seasonFactor * Math.sin(2 * Math.PI * dayOfYear / 365);
  return Math.round(biomeData.base * seasonFactor * (0.8 + 0.4 * Math.random()));
}
```
1. Add biome-specific yield tables with season factors
2. Implement seasonal variation using sinusoidal function
3. Add ±20% randomization to yields
4. Update all hunting-related tests

#### MED-03: Audio System Memory Leak
*Location:* `src/audio/ambiance.ts`
*Status:* Open
*Description:* Uncleaned `_tracks` Map causes memory growth over time.
*Impact:* Potential performance degradation during long play sessions.
*Evidence:* Memory growth observed in profiling (2026-06-23)
*Recommendations:*
```typescript
class AmbianceSystem {
  private _tracks: Map<string, Howl> = new Map();
  private _trackTTL: Map<string, number> = new Map();

  public unloadUnusedTracks(): void {
    const now = Date.now();
    this._tracks.forEach((track, key) => {
      if (now - (this._trackTTL.get(key) || 0) > 300000) { // 5 minute TTL
        track.unload();
        this._tracks.delete(key);
        this._trackTTL.delete(key);
      }
    });
  }
}
```
1. Implement track cleanup with 5-minute TTL
2. Add memory monitoring in development builds
3. Add test coverage for track lifecycle management
4. Document track management policies
*Implementation Priority:* Phase 1 (1 day)
