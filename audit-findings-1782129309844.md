# Frontier Project Audit Report - Consolidated Findings

## Executive Summary
The Frontier project maintains strong architectural foundations with excellent separation of concerns and adherence to the Three-Layer Safety Gate principle. This audit consolidates findings from multiple reports and triage sessions, identifying 4 critical/high severity issues requiring immediate attention, 10 medium severity issues needing resolution, and several low severity improvements. The project shows progress in some areas (RNG injection standardization, audio memory management) but requires focused effort on core gameplay mechanics, asset integrity, and system robustness.

## Current Status Summary
- **Resolved Issues**: 3 (RNG injection, audio memory leak, audio system tests)
- **Won't Fix**: 5 (intentional design decisions)
- **Confirmed Real Issues**: 19 requiring action
- **False Positives**: 2 (already addressed)

## Critical/High Severity Findings (4)

### 1. Visual Asset Corruption (High)
**Location**: `public/assets/sprites/horse_draft_base.png`, `horse_riding_base.png`
**Issue**: Sprite files contain malformed image data despite valid PNG headers
**Impact**: Broken visuals for core gameplay elements
**Recommendation**:
- Replace corrupted files with verified versions
- Implement asset validation during build process
- Add fallback sprites for missing/corrupted assets

### 2. Missing Narrative Thread Escalation (High)
**Location**: `src/engine/director.ts`
**Issue**: Director system doesn't advance narrative thread statuses
**Impact**: Reduced dynamic storytelling and player agency
**Recommendation**:
- Implement thread status advancement logic
- Define event-based triggers for thread progression
- Add test coverage for narrative escalation

### 3. Biome/Season Hunting Yield Variation Missing (High)
**Location**: `src/systems/supplies.ts`
**Issue**: Hunt yields use flat values without biome/season variation
**Impact**: Reduced game realism and strategic depth
**Recommendation**:
- Create `HUNTING_YIELD_BY_BIOME` table with biome-specific base yields
- Implement season factor calculation (e.g., `seasonFactor = 1 + 0.2 * sin(2π * dayOfYear/365)`)
- Add ±20% randomization to yields
- Update test coverage

### 4. River Crossing Terrain Lock Incomplete (High)
**Location**: `src/systems/movement.ts`, `src/types/game-state.ts`
**Issue**: Missing `riverCrossingInProgress` flag in `JourneyState`
**Impact**: River crossing encounters don't properly block movement
**Recommendation**:
- Add `riverCrossingInProgress: boolean` to `JourneyState`
- Modify movement system to return 0 distance when flag is set
- Clear flag on encounter resolution
- Add test coverage

## Medium Severity Findings (10)

### 1. Encounter Outcome Randomization Missing
**Location**: `src/systems/encounters.ts`
**Issue**: Numeric outcomes use fixed values instead of ±10% variation
**Impact**: Reduced replayability and realism
**Recommendation**:
- Add ±10% variation to numeric outcomes
- Log actual values in `EventRecord`
- Add test coverage

### 2. Companion-Specific Encounter Choices Missing
**Location**: `src/types/encounters.ts`, `src/systems/encounters.ts`
**Issue**: No `companion: CompanionId` requirement type for choices
**Impact**: Limited narrative interactions and companion utility
**Recommendation**:
- Add companion requirement type
- Implement choice availability filtering
- Add test case for "choice requiring Coe is unavailable if Coe deserted"

### 3. Pending Consequence Evaluation Incomplete
**Location**: `src/engine/director.ts`
**Issue**: Director doesn't parse `PendingConsequence.trigger` strings
**Impact**: Narrative consequences can't trigger based on game state
**Recommendation**:
- Implement trigger string parsing (e.g., `loyalty < 20`)
- Add state evaluation logic
- Implement consequence firing
- Add test coverage

### 4. Redundant State Snapshots
**Location**: `src/App.tsx` (agent bridge)
**Issue**: Full state snapshots pushed on every phase change
**Impact**: Performance overhead
**Recommendation**:
- Implement shallow comparison for state changes
- Only send changed slices to `/api/agent/state`

### 5. Missing Input Validation
**Location**: Agent bridge, external inputs
**Issue**: External inputs not consistently validated
**Impact**: Vulnerable to unexpected/malicious input
**Recommendation**:
- Validate all external inputs
- Implement type checking for agent commands
- Add input validation middleware

### 6. Missing Error Handling
**Location**: Agent bridge, audio system
**Issue**: Errors caught but not properly handled
**Recommendation**:
- Add proper error logging
- Implement graceful degradation
- Add error recovery paths

### 7. Type Safety Issues
**Location**: Agent bridge, audio system
**Issue**: Type assertions could fail at runtime
**Recommendation**:
- Replace with proper type guards
- Add runtime validation
- Implement type checking

### 8. Incomplete Companion Loyalty Momentum
**Location**: `src/systems/companions.ts`
**Issue**: Desertion probability doesn't account for loyalty+momentum combination
**Impact**: Companion desertion mechanics lack depth
**Recommendation**:
- Implement loyalty momentum calculation
- Add test coverage for desertion scenarios
- Document the momentum formula

### 9. Missing Audio Error Boundary
**Location**: `src/audio/ambiance.ts`
**Issue**: Missing error boundary for Howler.js initialization
**Recommendation**:
- Add try-catch around Howler initialization
- Implement graceful degradation
- Add test coverage for error handling

### 10. Test Coverage Gaps
**Areas needing expansion**:
- Companion system tests
- Agent bridge tests
- Health system (permanent debuffs)
- Movement system (detours)
