# Audit Findings

Date: 2026-06-21 (UTC)

# Frontier Project Audit Report

## Executive Summary
The Frontier project maintains a strong architectural foundation with good separation of concerns and adherence to the Three-Layer Safety Gate principle. This audit identified 1 critical severity issue, 6 high severity issues, and several medium/low severity findings that require attention. The project shows progress in addressing some previously identified issues, particularly in test coverage, but several core gameplay mechanics remain incomplete. Security and code quality improvements are needed to ensure long-term maintainability.

## Findings Summary

### Critical Severity (1)
1. **Missing Condition Escalation Logic** (Won't Fix)
   - The `ActiveCondition.daysUntilCritical` field exists but is intentionally not decremented to simplify gameplay in the initial release.
   - *Rationale*: Design decision to reduce complexity; may be addressed in future updates.

### High Severity (4/6 Remaining)
1. **Visual Asset Corruption**
   - Both `horse_draft_base.png` and `horse_riding_base.png` sprite files appear corrupted with malformed image data.
   - *Impact*: Broken visuals in core gameplay elements.

2. **Missing Narrative Thread Escalation**
   - The Director system lacks implementation for advancing narrative thread statuses based on game events.
   - *Impact*: Reduced dynamic storytelling and player impact on narrative.

3. **Incomplete Companion Loyalty Momentum**
   - Companion desertion logic remains simplistic; low loyalty combined with low morale does not increase desertion probability.
   - *Impact*: Reduced narrative depth and player consequence.

4. **Biome/Season Hunting Yield Variation Missing**
   - Hunt yields are flat values, lacking biome and seasonal variations.
   - *Impact*: Reduced game realism and strategic depth.

5. **River Crossing Terrain Lock Incomplete**
   - The `JourneyState` lacks a `riverCrossingInProgress` flag, preventing movement system from blocking progress during river crossings.
   - *Impact*: Breaks core gameplay mechanic for terrain-based movement restrictions.

6. **Inconsistent RNG Injection** (Resolved)
   - *Status*: Fixed in recent updates - RNG parameter now consistently handled across calculators.

### Medium Severity (10/12)
1. **Missing Audio Error Boundary**
   - The `src/audio/ambiance.ts` module lacks robust error handling for Howler.js initialization failures.
   - *Impact*: Potential game crashes if audio context fails.

2. **Audio Memory Leak**
   - The `_tracks` Map in `src/audio/ambiance.ts` never cleans up unused Howl instances.
   - *Impact*: Memory bloat during extended play sessions.

3. **Encounter Outcome Randomization Missing**
   - Numeric outcomes in encounters use fixed values instead of ±10% variation.
   - *Impact*: Reduced replayability and realism.

4. **Companion-Specific Encounter Choices Missing**
   - Encounter system lacks mechanism for choices requiring specific companions.
   - *Impact*: Limited narrative interactions and companion utility.

5. **Settlement Barter System Missing**
   - No "negotiate" choice implemented for settlement resupply encounters.
   - *Impact*: Reduced player agency in trade scenarios.

6. **Pending Consequence Evaluation Incomplete**
   - Director system doesn't parse `PendingConsequence.trigger` strings against game state.
   - *Impact*: Prevents dynamic narrative consequences.

7. **Hardcoded Warning Thresholds** (Won't Fix)
   - Supply warning thresholds remain hardcoded in `src/systems/supplies.ts`.
   - *Rationale*: Intentional for gameplay balance.

8. **Redundant State Snapshots**
   - Agent bridge pushes full game state snapshots on every phase change.
   - *Impact*: Performance overhead for large states.

9. **Inconsistent Rounding**
   - Different systems employ varying rounding approaches.
   - *Impact*: Potential floating-point precision issues.

10. **Missing Error Handling**
    - Several sections have empty or insufficient error handling.
    - *Impact*: Prevents graceful degradation.

11. **Type Safety Issues**
    - Codebase contains type assertions and loose typing in agent bridge.
    - *Impact*: Potential runtime type errors.

12. **Agent Bridge Security**
    - Development mode agent bridge lacks authentication/authorization.
    - *Impact*: Remote code execution risk if exposed.

### Low Severity (4)
1. **Visual Audit Manifest Outdated**
   - Manifest timestamp (2026-03-31) doesn't reflect current UI state.

2. **Missing UI Elements**
   - Several planned UI elements remain unimplemented:
     - Encounter outcome summary panel
     - Companion status display on map
     - Waypoint lore tooltips

3. **Audio System Limitations**
   - Sound effects system remains WIP; voiceover system unimplemented.

4. **Mobile Layout Responsiveness**
   - No documentation of mobile-specific UI considerations.

### Code Quality Issues (3/6 Remaining)
1. **Magic Numbers in Health System** (Won't Fix)
   - Hardcoded values remain intentional for gameplay balance.

2. **Complex Conditional Logic** (Won't Fix)
   - Deeply nested conditionals remain necessary for game mechanics.

3. **Documentation Gaps** (Won't Fix)
   - Acknowledged but not prioritized for current release.

## Recommendations

### Critical Priority
1. **Fix Corrupted Sprite Assets**
   - Replace `horse_draft_base.png` and `horse_riding_base.png` with uncorrupted versions.
   - Implement asset loading validation to detect future corruption.

### High Priority
1. **Implement Narrative Thread Escalation**
   - Add status advancement logic in Director system.
   - Define event-based triggers for thread progression.
   - *Location*: `src/engine/director.ts`

2. **Enhance Companion Desertion Logic**
   - Implement loyalty momentum calculation considering both loyalty and morale.
   - Integrate into desertion probability scaling.
   - *Location*: `src/systems/companions.ts`

3. **Implement Biome/Season Hunting Variation**
   - Create `HUNTING_YIELD_BY_BIOME` data table.
   - Implement date-based season factor calculation.
   - Add ±20% randomization to outcomes.
   - *Location*: `src/systems/supplies.ts`

4. **Complete River Crossing Implementation**
   - Add `riverCrossingInProgress: boolean` to `JourneyState`.
   - Modify movement system to return 0 distance when flag is active.
   - *Location*: `src/types/game-state.ts`, `src/systems/movement.ts`

### Medium Priority
1. **Add Audio Error Boundary**
   - Wrap Howler.js initialization with try-catch.
   - Implement graceful degradation for audio failures.
   - *Location*: `src/audio/ambiance.ts`

2. **Implement Audio Memory Cleanup**
   - Add `unloadUnusedTracks()` with 5-minute TTL.
   - Call during `stopAmbiance()`.
   - *Location*: `src/audio/ambiance.ts`

3. **Implement Encounter Outcome Randomization**
   - Add ±10% variation to numeric outcomes.
   - Log actual values in `EventRecord`.
   - *Location*: `src/systems/encounters.ts`

4. **Implement Companion-Specific Encounter Choices**
   - Add `companion: CompanionId` requirement to choice definitions.
   - Implement availability filtering.
   - *Location*: `src/types/encounters.ts`, `src/systems/encounters.ts`

5. **Optimize State Snapshots**
   - Implement shallow comparison for agent bridge.
   - Only send changed state slices.
   - *Location*: `src/App.tsx`

6. **Standardize Rounding Approach**
   - Define consistent rounding strategy.
   - Implement utility function if needed.
   - Apply uniformly across systems.

7. **Improve Error Handling**
   - Replace empty catch blocks with proper logging.
   - Implement graceful degradation paths.

8. **Enhance Type Safety**
   - Replace type assertions with proper type guards.
   - Strengthen validation for agent bridge commands.

9. **Secure Agent Bridge**
   - Implement authentication for commands.
   - Add command validation.
   - Restrict to development environment only.

### Low Priority
1. **Update Visual Audit Manifest**
   - Capture current screenshots of all UI states.
   - Update manifest timestamp.

2. **Implement Missing UI Elements**
   - Develop encounter outcome summary panel.
   - Add companion status display on map.
   - Implement waypoint lore tooltips.

3. **Complete Audio System**
   - Finalize sound effects system.
   - Implement voiceover system.

4. **Improve Mobile Responsiveness**
   - Document mobile-specific UI considerations.
   - Conduct mobile layout tests.

## Test Coverage Recommendations
1. **Expand Audio System Tests**
   - Add tests for crossfade cancellation.

   - Test error handling for initialization failures.

2. **Add Agent Bridge Tests**
   - Test command validation.

3. **Maintain Existing Test Coverage**

The project remains well-positioned for successful completion with these improvements.
