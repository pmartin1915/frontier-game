# Audit Findings

**Date:** 2026-07-18 (UTC)

## Overview
This audit reviews the "frontier" project's automated audit system, focusing on persistent issues in the `mistral-large-latest` model's output and the triage process. The analysis reveals critical flaws in the audit model's contextual understanding, leading to self-referential meta-issues, stale findings, and ambiguous triage rationales. These issues significantly degrade the audit pipeline's reliability and efficiency.

---

## Findings

### 1. CRITICAL: Persistent Self-Referential Meta-Issues and Misattribution
**Category:** Bug / Process Reliability
**Severity:** Critical

#### Description
The `mistral-large-latest` model continues to generate "CRITICAL" findings that are meta-comments about the audit process itself, rather than actual code defects. These findings:
- Falsely claim a "100% false-positive rate for meta-issues" within a "triage system."
- Repeatedly misattribute these issues to `src/engine/director.ts`, despite triage rationales confirming this file contains only deterministic narrative-generation logic.
- Demonstrate a fundamental misunderstanding of the audit system's architecture and the roles of its components.

#### Impact
- Wastes triage resources on irrelevant findings.
- Undermines confidence in the audit system's ability to identify genuine defects.
- Obscures real issues with noise, reducing audit effectiveness.

#### Evidence
- **Triage Reports:**
  - `triage-frontier-audit-20260717111413.md` (Fingerprint: `23256f2b49d3…`): Labeled `false-positive` with rationale confirming no defects in `director.ts`.
  - `triage-frontier-audit-20260717173346.md` (Fingerprint: `46b8ba2bd362…`): Labeled `false-positive`; cites `*` as an issue with no actual defect.
  - `triage-frontier-audit-20260717193243.md` (Fingerprint: `46b8ba2bd362…`): Duplicate of above, confirming the pattern.
- **Prior Audit Reports:**
  - `audit-findings-1784277244632.md` and `audit-findings-1784313203180.md` document identical issues.

#### Recommendations
- **R1.1:** Retrain or reconfigure the audit model to exclude meta-comments about the audit process itself.
- **R1.2:** Implement guardrails to prevent misattribution of issues to `director.ts` or other core components.
- **R1.3:** Add a post-processing filter to suppress findings that reference the audit system rather than the target codebase.

---

### 2. HIGH: Generation of Stale Findings for Non-Existent Files
**Category:** Bug / Data Quality
**Severity:** High

#### Description
The audit model occasionally generates findings for files or paths that no longer exist in the codebase. These findings are labeled `false-positive` or `stale` during triage, indicating:
- The model may be operating on an outdated snapshot of the codebase.
- The model may hallucinate file paths.

#### Impact
- Wastes triage resources on unverifiable findings.
- Reduces audit efficiency and accuracy.
- Causes confusion for developers.

#### Evidence
- **Triage Reports:**
  - `triage-frontier-audit-20260716153334.md`: Labeled `false-positive` with rationale stating the referenced file no longer exists.
- **Prior Audit Reports:**
  - `audit-findings-1784277244632.md` and `audit-findings-1784313203180.md` document similar issues.

#### Recommendations
- **R2.1:** Ensure the audit model always operates on the latest codebase snapshot.
- **R2.2:** Implement pre-validation to verify file existence before generating findings.
- **R2.3:** Automatically mark findings as `stale` if referenced files are deleted.

---

### 3. MEDIUM: Ambiguous Triage Rationales and Issue Descriptions
**Category:** Process Improvement / Code Quality
**Severity:** Medium

#### Description
Triage reports exhibit two key ambiguities:
1. **Fingerprint-Only Issue Descriptions:** Some reports list raw fingerprints (e.g., `23256f2b49d3…`) as the "Issue," requiring manual lookup to understand the original finding.
2. **Misleading Rationales:** Rationales for meta-issues (e.g., "file no longer exists") can mislead reviewers into thinking core files like `director.ts` were deleted, when the issue is actually about a referenced finding.

#### Impact
- Reduces clarity and readability of triage reports.
- Increases friction for human reviewers.
- Risks misinterpretation of triage decisions.

#### Evidence
- **Triage Reports:**
  - `triage-frontier-audit-20260717111413.md`: Lists fingerprint `23256f2b49d3…` as the issue.
  - `triage-frontier-audit-20260716153334.md`: Rationale ambiguously states "file no longer exists" for a meta-issue.

#### Recommendations
- **R3.1:** Ensure triage reports display the original finding's descriptive text, not just fingerprints.
- **R3.2:** Refine rationale generation to avoid misleading statements about core files.
- **R3.3:** Standardize labels (e.g., use `stale` for non-existent files, not `false-positive`).
