# Audit Findings

```
Date: 2026-06-20

## Audit Report: Frontier Project

### 1. Security Findings
**Severity: None**
- No critical security vulnerabilities detected in the reviewed codebase.
- Agent bridge in `App.tsx` is properly guarded by `import.meta.env.DEV` and uses HTTPS POST.
- No sensitive data exposure in state snapshots or command payloads.

### 2. Code Quality Findings

#### 2.1. High Severity
**Finding: Incomplete Condition Escalation Implementation**
- **Location**: `src/systems/health.ts`
- **Issue**: `ActiveCondition.daysUntilCritical` exists but is never decremented, preventing condition escalation logic from functioning.
- **Impact**: Core game mechanic (untreated conditions worsening) is non-functional.
- **Recommendation**: Implement daily decrement in `calculateHealth()` and emit `HealthEvent` when `daysUntilCritical` reaches 0.

#### 2.2. Medium Severity
**Finding: Missing Audio Error Boundary**
- **Location**: `src/audio/ambiance.ts`
- **Issue**: No error boundary for Howler.js initialization failures, which could crash the game if audio context fails.
- **Recommendation**: Wrap Howler initialization in try-catch and provide fallback silent operation.

**Finding: Unused Companion Loyalty Momentum**
- **Location**: `ROADMAP.md` (Phase 1, item 3)
- **Issue**: Companion desertion probability based on loyalty/morale is not implemented in `src/systems/companions.ts`.
- **Recommendation**: Add loyalty-dependent desertion roll in `processCompanionDay()`.

### 3. Bug Findings

#### 3.1. High Severity
**Finding: River Crossing Terrain Lock Not Implemented**
- **Location**: `ROADMAP.md` (Phase 2, item 1)
- **Issue**: `JourneyState.riverCrossingInProgress` flag does not exist, preventing terrain-based movement blocking.
- **Impact**: River encounters cannot block forward movement as designed.
- **Recommendation**: Add flag to `src/types/game-state.ts` and modify `calculateMovement()` to return 0 distance when active.

#### 3.2. Medium Severity
**Finding: Audio Memory Leak**
- **Location**: `src/audio/ambiance.ts`
- **Issue**: `_tracks` Map never cleans up unused Howl instances, causing memory bloat during long sessions.
- **Recommendation**: Add `unloadUnusedTracks()` with 5-minute TTL and call during `stopAmbiance()`.

### 4. Code Health Findings

#### 4.1. High Severity
**Finding: Inconsistent RNG Injection**
- **Location**: `src/systems/movement.ts`, `src/systems/supplies.ts`
- **Issue**: `rng` parameter is optional in some calculators but required in others, causing test complexity.
- **Recommendation**: Standardize all calculators to accept optional `rng` parameter with `Math.random` fallback.

#### 4.2. Medium Severity
**Finding: Hardcoded Warning Thresholds**
- **Location**: `src/systems/supplies.ts`
- **Issue**: `WARNING_LOW` (5) and `WARNING_CRITICAL` (2) are hardcoded, preventing UI customization.
- **Recommendation**: Move to `src/types/game-state.ts` as `SUPPLY_WARNING_THRESHOLDS`.

### 5. Test Coverage Gaps

#### 5.1. High Severity
**Finding: Missing Condition Escalation Tests**
- **Location**: `tests/systems/health.test.ts`
- **Issue**: No tests for `daysUntilCritical` decrement or permanent debuff application.
- **Recommendation**: Add test cases:
  ```typescript
  it('decrements daysUntilCritical each day', () => { ... });
  it('applies permanent debuff when daysUntilCritical reaches 0', () => { ... });
  ```

#### 5.2. Medium Severity
**Finding: Incomplete Audio Test Coverage**
- **Location**: `src/audio/README.md`
- **Issue**: No tests for crossfade cancellation or memory management.
- **Recommendation**: Add tests for:
  - Rapid track switching
  - Track unloading after TTL
  - Error handling for missing files

### 6. Performance Findings

#### 6.1. Medium Severity
**Finding: Redundant State Snapshots**
- **Location**: `src/App.tsx`
- **Issue**: Agent bridge pushes full state snapshots on every phase change, causing unnecessary serialization.
- **Recommendation**: Use shallow comparison to only send changed slices.

### 7. Recommendations

#### 7.1. Immediate Actions
1. **Implement Condition Escalation** (ROADMAP Phase 1, item 2):
   - Modify `calculateHealth()` to decrement `daysUntilCritical`
   - Add permanent debuff logic (e.g., `lastingMovementPenalty`)
   - Extend `HealthEvent` with `worsened` type

2. **Add River Crossing Terrain Lock**:
   - Add `riverCrossingInProgress: boolean` to `JourneyState`
   - Modify `calculateMovement()` to return 0 distance when active
   - Add test case in `tests/systems/encounters.test.ts`

#### 7.2. Follow-up Actions
1. **Companion Loyalty Momentum**:
   - Add desertion probability calculation in `processCompanionDay()`
   - Test with low loyalty/morale scenarios

2. **Audio System Improvements**:
   - Add memory cleanup for unused tracks
   - Implement error boundary for Howler initialization

3. **Standardize RNG Injection**:
   - Update all calculators to accept optional `rng` parameter
   - Add default fallback to `Math.random`

### 8. Positive Observations
- Excellent separation of concerns (Three-Layer Safety Gate)
- Comprehensive test coverage for existing features
- Clean TypeScript interfaces with clear documentation
- Well-structured audio system with volume normalization
- Effective use of Zustand for state management

### 9. Risk Assessment
| Finding | Severity | Risk | Mitigation Priority |
|---------|----------|------|---------------------|
| Condition Escalation Missing | High | Critical game mechanic broken | 1 |
| River Crossing Terrain Lock | High | Core encounter design broken | 2 |
| Companion Loyalty Momentum | Medium | Narrative depth reduced | 3 |
| Audio Memory Leak | Medium | Long-session crashes | 4 |
```
