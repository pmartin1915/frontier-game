# Audit Findings

# Frontier Project Audit Report

## Executive Summary
The Frontier project maintains a robust architectural foundation with strong separation of concerns and adherence to the Three-Layer Safety Gate principle. This audit consolidates findings from previous reports, updates statuses, and provides actionable recommendations. While core systems are well-structured, several critical and high-severity issues persist, particularly in game mechanics completeness, asset integrity, and system robustness. Code quality and security remain areas requiring focused improvement to ensure long-term maintainability.

## Findings Summary

### Critical Severity (1)
1. **Missing Condition Escalation Logic** (Won't Fix)
   - The `ActiveCondition.daysUntilCritical` field exists but remains undecremented, preventing health conditions from worsening and applying permanent debuffs.
   - *Rationale*: Design decision to simplify escalation paths for initial release.

### High Severity (5)
1. **Visual Asset Corruption**
   - `horse_draft_base.png` and `horse_riding_base.png` sprite files remain corrupted with malformed image data.
   - *Impact*: Broken visuals in core gameplay elements.

2. **Missing Narrative Thread Escalation**
   - Director system lacks implementation for advancing narrative thread statuses based on game events.
   - *Impact*: Reduced dynamic storytelling and player agency.

3. **Incomplete Companion Loyalty Momentum**
   - Companion desertion logic remains simplistic; low loyalty + low morale does not increase desertion probability.
   - *Impact*: Reduced narrative depth and player consequence.

4. **Biome/Season Hunting Yield Variation Missing**
   - Hunt yields remain flat values without biome/seasonal variations.
   - *Impact*: Reduced game realism and strategic depth.

5. **River Crossing Terrain Lock Incomplete**
   - `JourneyState` lacks `riverCrossingInProgress` flag, preventing movement system from blocking progress during river crossings.
   - *Impact*: Broken terrain-based movement mechanics.

### Medium Severity (10)
1. **Missing Audio Error Boundary**
   - `src/audio/ambiance.ts` lacks robust error boundary for Howler.js initialization failures.
   - *Risk*: Potential game crashes if audio context fails.

2. **Audio Memory Leak**
   - `_tracks` Map in `src/audio/ambiance.ts` never cleans up unused Howl instances.
   - *Risk*: Memory bloat during extended play sessions.

3. **Encounter Outcome Randomization Missing**
   - Numeric outcomes in encounters use fixed values instead of ±10% variation.
   - *Impact*: Reduced replayability and realism.

4. **Companion-Specific Encounter Choices Missing**
   - Encounter choice system lacks mechanism for companion-specific requirements.
   - *Impact*: Limited narrative interactions and companion utility.

5. **Settlement Barter System Missing**
   - No "negotiate" choice implemented for settlement resupply encounters.
   - *Impact*: Reduced player agency in trade scenarios.

6. **Pending Consequence Evaluation Incomplete**
   - Director system does not parse `PendingConsequence.trigger` strings against game state.
   - *Impact*: Prevents dynamic narrative consequences.

7. **Inconsistent RNG Injection**
   - `rng` parameter remains inconsistently handled across calculators.
   - *Impact*: Complicates testing and standardization.

8. **Inconsistent Rounding**
   - Different systems employ varying rounding approaches.
   - *Risk*: Floating-point precision issues and calculation inconsistencies.

9. **Agent Bridge Security**
   - Development mode agent bridge lacks authentication/authorization.
   - *Risk*: Remote code execution if bridge is exposed.

10. **Missing Input Validation**
    - External inputs (agent bridge commands) not consistently validated.
    - *Risk*: Vulnerable to unexpected/malicious input.

### Low Severity (4)
1. **Visual Audit Manifest Outdated**
   - Manifest timestamp (2026-03-31) does not reflect current UI state.

2. **Missing UI Elements**
   - Encounter outcome summary panel, companion status display, and waypoint lore tooltips remain unimplemented.

3. **Audio System Limitations**
   - Sound effects system remains WIP; voiceover system unimplemented.

4. **Mobile Layout Responsiveness**
   - No documentation of mobile-specific UI considerations.

### Code Quality Issues (4)
1. **Magic Numbers in Health System** (Won't Fix)
   - Hardcoded values (e.g., `DEHYDRATION_HEALTH_PENALTY = -5`) remain in `src/systems/health.ts`.
   - *Rationale*: Intentional for gameplay balance.

2. **Complex Conditional Logic** (Won't Fix)
   - Deeply nested conditionals persist in health recovery and movement systems.
   - *Rationale*: Required for game mechanics complexity.

3. **Missing Error Handling**
   - Empty/incomplete error handling blocks in agent bridge and audio system.

4. **Type Safety Issues**
   - Type assertions (e.g., `as unknown as Phaser.Tweens.Tween`) and loose typing in agent bridge.

## Recommendations

### Critical Priority
*None* (Won't Fix for condition escalation)

### High Priority
1. **Fix Corrupted Sprite Assets**
   - Replace `horse_draft_base.png` and `horse_riding_base.png` with uncorrupted versions.
   - Implement asset loading validation.
   - *Location*: `public/assets/sprites/`

2. **Implement Narrative Thread Escalation**
   - Add `NarrativeThread` status advancement logic in Director system.
   - Define event-based triggers for thread progression.
   - *Location*: `src/engine/director.ts`

3. **Enhance Companion Desertion Logic**
   - Implement loyalty momentum calculation considering loyalty + morale.
   - Integrate into desertion probability scaling.
   - *Location*: `src/systems/companions.ts`

4. **Implement Biome/Season Hunting Variation**
   - Create `HUNTING_YIELD_BY_BIOME` data table.
   - Implement date-based season factor calculation.
   - Add ±20% randomization to outcomes.
   - *Location*: `src/systems/supplies.ts`

5. **Complete River Crossing Implementation**
   - Add `riverCrossingInProgress: boolean` to `JourneyState`.
   - Modify movement system to return 0 distance when flag is active.
   - *Location*: `src/types/game-state.ts`, `src/systems/movement.ts`

### Medium Priority
1. **Add Audio Error Boundary**
   - Wrap Howler.js initialization with try-catch.
   - Implement graceful degradation.
   - *Location*: `src/audio/ambiance.ts`

2. **Implement Audio Memory Cleanup**
   - Add `unloadUnusedTracks()` with 5-minute TTL.
   - Call during `stopAmbiance()`.
   - *Location*: `src/audio/ambiance.ts`

3. **Implement Encounter Outcome Randomization**
   - Add ±10% variation to numeric outcomes.
   - Log actual values in `EventRecord`.
   - *Location*: `src/systems/encounters.ts`

4. **Implement Companion-Specific Encounter Choices**
   - Add `companion: CompanionId` requirement to choice definitions.
   - Implement choice filtering based on companion presence.
   - *Location*: `src/types/encounters.ts`, `src/systems/encounters.ts`

5. **Standardize RNG Injection**
   - Refactor calculators to accept optional `rng` parameter.
   - Provide `Math.random` fallback.
   - *Location*: `src/systems/movement.ts`, `src/systems/supplies.ts`

6. **Improve Agent Bridge Security**
   - Implement authentication (API keys/session tokens).
   - Add command validation.
   - Restrict to development environment.
   - *Location*: `src/App.tsx`, `src/api/`

7. **Add Input Validation**
   - Implement comprehensive validation for agent bridge commands.
   - Perform type checking/sanitization.
   - *Location*: `src/api/`

### Low Priority
1. **Update Visual Audit Manifest**
   - Capture current UI screenshots.
   - Update manifest timestamp.

2. **Implement Missing UI Elements**
   - Develop encounter outcome summary panel.
   - Add companion status display on map.
   - Implement waypoint lore tooltips.

3. **Complete Audio System**
   - Finalize sound effects system.
   - Implement voiceover system.

4. **Improve Mobile Responsiveness**
   - Document mobile-specific UI considerations.
   - Conduct mobile layout tests.

### Code Quality Improvements
1. **Improve Error Handling**
   - Replace empty catch blocks with proper logging.
   - Implement graceful degradation paths.

2. **Enhance Type Safety**
   - Replace type assertions with proper type guards.
   - Strengthen type checking for external inputs.

## Test Coverage Recommendations

   - Verify memory management (track unloading after TTL).
   - Test error handling for Howler.js initialization failures.

2. **Agent Bridge Tests**
